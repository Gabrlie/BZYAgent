# 测试策略与质量保证

## 测试范围
- 后端 API 功能与权限验证
- AI 生成流程与 SSE 进度
- 文档上传与下载
- 前端页面核心流程

## 建议的测试类型
- 单元测试
- 认证与权限校验
- 课程与文档 CRUD
- 排课计算逻辑
- 集成测试
- 授课计划生成完整链路
- 教案生成完整链路
- AI 对话流式返回
- 端到端测试
- 登录到课程生成的核心路径

## 关键测试用例
- 无 Token 访问受保护接口应返回 401。
- Base URL 未按规则配置时应返回 400。
- 文件上传超限或类型非法应返回 400。
- 课程目录为空时授课计划生成应中止并返回错误。
- 授课计划缺失时教案生成应中止并返回错误。

## 质量门禁建议
- 新增功能必须补充对应 API 测试。
- SSE 生成流程至少具备一次集成测试。
- 文档模板变更需验证模板字段完整性。
